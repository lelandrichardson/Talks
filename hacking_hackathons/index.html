Splitting Up Development

You *DO* want to optimize your time. You can do this by splitting up development effectively.
This can be a lot easier if you follow an API Design pattern for a "client" to consume.

Mock Client/Server Behavior

  • Use HTTP Debuggers to emulate Client Requests
  • Chrome Plugin - REST Console
  • Fiddler HTTP Debugger
  • Create mock .json files to consume from javascript
$.ajax({ /* GET Example */
    url: "/mock/patient_get.json", /*TODO: becomes "/patients/{id}"*/
    type: "GET",
    success: function(patient){/* ... */},
    error: function(patient){ /* ... */}
});

$.ajax({ /* PUT Example */
    url: "/mock/patient_put.json", /*TODO: becomes "/patients"*/
    type: "GET", /*TODO: becomes "PUT"*/
    success: function(patient){/* ... */},
    error: function(patient){ /* ... */}
});
Server Side Devs: Get familiar with Fiddler and a HTTP/REST console to emulate client calls.
Client Side Guys: Create mock .json files and make ajax calls to them instead.

Data Storage

To SQL or to NoSQL?


Again: Just stick with what you know.

Multiple Choice:
Facebook, Twitter, Wikipedia, YouTube, Slashdot, Google Analytics...
  • a.) were all built using MySQL
  • b.) were all built using NoSQL Technologies

What can we learn from them?
  • SQL is actually very fast
  • NoSQL/SQL both have advantages/disadvantages
  • Neither are Turn-Key Solutions
  • ACID vs BASE

Relational


  • PostgreSQL
  • MySQL
  • MS-SQL
Honestly, they are all pretty solid and robust.
Very similar syntaxes
Personally, I choose PostgreSQL, however if you are trying to move quickly, the MySQL community is far and away the largest, and you will be able to find the most resources and support for this.

NoSQL

Document Store:
  • CouchDB
  • MongoDB

Column Store:
  • Hadoop / HBase
  • Cassandra

Key/Value Store:
  • Redis
  • Riak
Document Store: High Writes, Low Reads, Some Analysis.
Column Store: Data stored as columns. can be very fast for stats/aggregates. Hard to get right.
KV Store: High Reads, High Writes, Simple Queries, No Analysis.

Require.JS

  • Asynchronous Module Definition (AMD)
  • Keep track of dependencies intelligently
  • Write Code in Modules
  • Prevents Global Namespace Pollution
  • requirejs.org

Require.JS

Defining Modules
//Explicitly defines the "foo/module" module:
                define("foo/module", ["jquery", "foo/other"],
                function($, other) {
                //Define foo/module object in here.
                // jQuery '$' and 'other' modules are set as dependencies
                }
                );

Require.JS

Application Code
// Explicitly injects dependent modules
                require(["jquery","foo/other","foo/module"],
                function($,other,module){
                //write page-level logic here
                module.start();
                other.bind("#someElementId");
                $("#someElementId").animate(/* ... */);
                // ...
                }
                );
